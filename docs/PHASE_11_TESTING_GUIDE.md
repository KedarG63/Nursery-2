# Phase 11 Testing Guide
## Quick Start for Testing Product & Inventory Management

---

## Prerequisites

1. **Database:** PostgreSQL running with all migrations applied
2. **Backend:** Node.js v18+ installed
3. **Frontend:** Node.js v18+ installed
4. **Environment:** `.env` files configured

---

## Quick Start Commands

### 1. Start Backend
```bash
cd backend
npm install
npm run dev
```
Backend should start on `http://localhost:5000`

### 2. Start Frontend
```bash
cd frontend
npm install
npm run dev
```
Frontend should start on `http://localhost:5173`

## Testing Products (Issue #49 & #50)

### Access the Products Page
1. Login to the application
2. Navigate to `/products` or click "Products" in sidebar
3. You should see the Products list page

### Test Search & Filters
1. **Search:** Type in search box → Results filter after 500ms
2. **Category Filter:** Select "Flowering" → Only flowering products show
3. **Status Filter:** Select "Active" → Only active products show
4. **Pagination:** Click page numbers → Data updates

### Test Create Product
1. Click "Add Product" button (Admin/Manager only)
2. Fill in form:
   - Name: "Red Rose"
   - Category: "Flowering"
   - Growth Period: 45
   - Description: "Beautiful red roses"
   - Image: Upload a JPG/PNG (optional)
3. Click "Create Product"
4. Should see success toast
5. Product appears in list

### Test Edit Product
1. Click edit icon (pencil) on any product
2. Modify fields
3. Click "Update Product"
4. Should see success toast

### Test Delete Product
1. Click delete icon (trash) on any product
2. Confirm deletion
3. Product removed from list

### Expected Behaviors
- ✅ Search debounces (waits 500ms)
- ✅ Filters work independently and together
- ✅ Add button hidden for non-admin users
- ✅ Image preview shows after upload
- ✅ Form validation errors display
- ✅ Loading states show during operations

## Testing SKUs (Issue #51)

### Access the SKUs Page
1. Navigate to `/skus`
2. You should see the SKUs list page

### Test Create SKU
1. Click "Add SKU" button (Admin/Manager only)
2. Fill in form:
   - Product: Search and select "Red Rose"
   - Size: "Medium"
   - Pot Type: "Plastic"
   - SKU Code: Auto-generated (e.g., "REDR-Medium-Plastic")
   - Unit Price: 150
   - Cost Price: 100
   - Min Stock Level: 10
3. Click "Create SKU"
4. Should see success toast
5. SKU appears in list

### Test Filters
1. **Product Filter:** Select a product → Only SKUs for that product show
2. **Search:** Type SKU code → Results filter
3. **Low Stock:** Check "Show Low Stock" → Only low stock SKUs show

### Test Stock Indicators
1. Look at "Current Stock" column
2. Colors should be:
   - 🟢 Green: Stock > Min Stock
   - 🟡 Yellow: 0 < Stock ≤ Min Stock
   - 🔴 Red: Stock = 0

### Expected Behaviors
- ✅ SKU code auto-generates from selections
- ✅ Validation: Unit Price must be > Cost Price
- ✅ Product dropdown is searchable
- ✅ Stock colors update correctly
- ✅ Cost price hidden from non-admin users

## Testing Lots Inventory (Issue #52)

### Access the Lots Page
1. Navigate to `/inventory/lots`
2. You should see the Lots list page

### Test Create Lot
1. Click "Create Lot" button (Admin/Manager only)
2. Fill in form:
   - SKU: Search and select a SKU
   - Quantity: 50
   - Location: "A-1-01"
   - Stage: "seed"
   - Notes: "First batch of roses"
3. Click "Create Lot"
4. Should see success toast
5. Lot appears in list with auto-generated lot number (e.g., LOT-2024-001)

### Test QR Code View
1. Click "QR Code" button (eye icon) on any lot
2. Modal opens showing QR code image
3. Test "Download PNG" button → QR code downloads
4. Test "Print" button → Print dialog opens
5. Close modal

### Test Stage Update
1. Find the Stage dropdown in table row
2. Select a different stage (e.g., "germination")
3. Stage updates immediately
4. Toast notification shows

### Test Location Change
1. Click "Move Location" button (location icon)
2. Dialog opens
3. Enter new location: "B-2-05"
4. Add notes: "Moved to new greenhouse"
5. Click "Confirm"
6. Location updates in table

### Test Filters
1. **Stage Filter:** Select "ready" → Only ready lots show
2. **Location Filter:** Select a location → Only lots at that location show
3. **SKU Filter:** Select a SKU → Only lots for that SKU show
4. **Show Overdue:** Check checkbox → Only overdue lots show

### Expected Behaviors
- ✅ Lot number auto-generated by backend
- ✅ Expected ready date calculates correctly
- ✅ Stage chips have correct colors
- ✅ QR code displays and downloads
- ✅ Location format validated (A-1-01 pattern)
- ✅ Overdue lots highlighted

## Testing Mobile Scanner (Issue #53)

### Access the Scanner Page
1. Navigate to `/inventory/lots/scan`
2. You should see full-screen scanner interface

### Test Camera Scanning (Mobile Device Recommended)
1. Grant camera permission when prompted
2. Point camera at a lot QR code
3. QR code auto-detects and scans
4. Phone vibrates (200ms)
5. Bottom drawer slides up showing lot details

### Test Manual Input
1. Click "Enter Manually" button
2. Type lot number manually
3. Press Enter or click "Scan"
4. Lot details display in drawer

### Test Quick Actions
1. After scanning a lot, bottom drawer shows details
2. Test stage transition buttons:
   - Click "Mark as Germination" (or next valid stage)
   - Stage updates
   - Toast notification shows
   - Vibration feedback
3. Test "Move Location" button
4. Test "View Full Details" link
5. Click "Scan Another" to scan next lot

### Test Scan History
1. Click history icon in top bar
2. Side drawer opens showing last 10 scans
3. Click on a history item → Loads that lot
4. Click "Clear History" → Confirms and clears

### Test Camera Controls
1. **Switch Camera:** Click camera switch icon → Toggles front/back
2. **Manual Toggle:** Toggle between camera and manual input

### Expected Behaviors
- ✅ Camera permission requested on mount
- ✅ QR codes detected within 1 second
- ✅ Vibration feedback works (mobile only)
- ✅ History persists in localStorage
- ✅ Valid stage transitions only shown
- ✅ Drawer swipeable on mobile
- ✅ Works in portrait and landscape

## Error Handling Testing

### Test Network Errors
1. Stop backend server
2. Try to load products page
3. Should show error toast: "Failed to load products"
4. Restart backend
5. Retry → Should work

### Test Validation Errors
1. Try to create product with empty name
2. Should show error: "Name is required"
3. Try to create SKU with Unit Price < Cost Price
4. Should show error: "Unit price must be greater than cost price"

### Test Permission Errors
1. Login as Sales user
2. Try to access `/inventory/lots` directly via URL
3. Should show access denied or redirect

### Test File Upload Errors
1. Try to upload 10MB image
2. Should show error: "File size too large"
3. Try to upload .pdf file
4. Should show error: "Invalid file type"

## Common Issues & Solutions

### Issue: Camera not working on mobile
**Solution:** Ensure app is accessed via HTTPS (required for camera API)

### Issue: QR code not detected
**Solution:** Ensure QR code is well-lit and within the 250x250px box

### Issue: "Failed to load products" error
**Solution:** Check backend is running and database is connected

### Issue: Image upload fails
**Solution:** Check `backend/uploads/products/` directory exists and has write permissions

### Issue: Filters not working
**Solution:** Check network tab for API errors, verify query params

### Issue: Role-based buttons not hiding
**Solution:** Check Redux state has correct user role from login
